Introdução

O LilyGO T-Deck Plus é um dispositivo standalone completo para Meshtastic com ecrã, teclado QWERTY, GPS integrado e LoRa. Este guia cobre desde a instalação inicial até configuração avançada.
Características principais: - ESP32-S3 + SX1262 LoRa - GPS L76K integrado - Ecrã TFT 2.8" touchscreen - Teclado físico QWERTY - Bateria 18650 (não incluída) - MicroSD para mapas offline


Requisitos
Hardware

LilyGO T-Deck Plus
Cabo USB-C (dados, não apenas carga)
Bateria 18650 (recomendado: 3000mAh+)
Antena LoRa 868MHz (Europa) ou 915MHz (US)
MicroSD card FAT32 ≤32GB (opcional, para mapas)

Software

Browser Chromium (Chrome, Edge, Brave)
Internet ativa
Meshtastic App (Android/iOS) para pairing Bluetooth

Avisos Importantes

Nunca ligue o dispositivo sem antena conectada - pode danificar o módulo LoRa Usa sempre cabo USB-C com suporte de dados (não apenas carga)
Define região correta (EU_868 para Portugal) para cumprir regulamentação ANACOM


Modo Programming (DFU/Flash)

Para flashear firmware ou fazer erase, o T-Deck Plus precisa entrar em modo DFU.

Procedimento

Desliga o T-Deck (switch lateral OFF)
Pressiona e mantém o trackball (botão central)
Enquanto pressiona trackball, liga o T-Deck (switch ON)
Aguarda 2-3 segundos mantendo trackball pressionado
Liberta o trackball
Conecta cabo USB-C ao computador

Confirmação de Modo DFU

Ecrã do T-Deck fica completamente negro (sem backlight)
No computador aparece dispositivo: "USB JTAG/serial debug unit" ou "ESP32-S3"

Nota: Se não funcionar à primeira, repete o processo. Alguns cabos USB-C to USB-C podem causar problemas - tenta USB-A to USB-C.


Instalação Firmware - Erase + Flash

Quando usar: Primeira instalação, dispositivo com problemas graves, mudança de firmware type (ex: alpha → beta).

Passo a Passo

Entra em Modo Programming

Segue procedimento da secção anterior.

Acede ao Web Flasher

Abre browser Chromium
Vai a: https://flasher.meshtastic.org

Seleciona Dispositivo

Clica "Select Target Device"
Escolhe: "LILYGO T-Deck" (funciona para T-Deck Plus)

Escolhe Versão Firmware

Recomendado: Seleciona versão BETA mais recente (evita alpha em produção)
Exemplo:	ou superior
Não uses versões alpha para uso diário

Configura Full Erase

 Ativa opção: "Full Erase and Install"
Isto apaga toda flash incluindo configurações antigas

Conecta Dispositivo

Clica "Connect"
Seleciona porta: "USB JTAG/serial debug unit"
Clica "Connect"

Flash

Clica "Flash" e depois "CONTINUE" Aguarda conclusão (1-3 minutos)
NÃO desconectes USB durante processo

Finalização

Quando vires "Flash Complete", fecha browser
Desliga T-Deck (switch OFF)
Liga novamente (switch ON)
Dispositivo arranca com firmware fresco e MUI (Meshtastic UI)

Web Flasher

Update (Sem Erase)

Flash e Reboot

Configuração Inicial

Após flash inicial, configura parâmetros essenciais.

Via Meshtastic Web Client

Conecta via USB Serial

Vai a: https://client.meshtastic.org Clica "Connect" → "USB Serial" Seleciona porta do T-Deck

Define Região LoRa

Menu lateral: "LoRa"
Region: Seleciona EU_868 (Portugal/Europa) Clica "Save"
 Obrigatório em Portugal: EU_868 (868MHz, 500mW, 10% duty cycle) conforme ANACOM.

Configura Posição GPS

Menu: "Position"
GPS Mode: "Enabled"
GPS Update Interval: 120s (ajusta conforme necessidade) Position Broadcast: Ativa se quiseres partilhar localização Clica "Save"

Node Info (Opcional)

Menu: "Settings" → "Device"
Node Name: Define nome do teu node
Role: CLIENT (uso normal) ou ROUTER (node fixo)

Canais (Opcional)

Menu: "Channels"
Configura canais adicionais ou junta-te a mesh existente
Primary Channel vem pré-configurado

Via MUI no T-Deck

Após configuração inicial via web/app, podes ajustar no próprio T-Deck: - Settings → LoRa → Verifica região - Settings → Position → GPS status - Map →
Visualiza nodes e posição GPS

Bluetooth - Modo Programming

Quando usar: Pairing inicial com smartphone, configuração via app, troubleshooting.

Activar Bluetooth Programming Mode

Via MUI (Firmware 2.6+)

No T-Deck, vai a: Settings (ícone engrenagem)
Seleciona: "Reboot/Shutdown"
Clica no ícone Bluetooth (centro do ecrã)
Ecrã mostra: ">> Programming Mode <<" com passcode (ex: 123456)

Pairing com Smartphone

Abre Meshtastic App (Android/iOS)
Vai a Bluetooth → Scan
Seleciona teu T-Deck (nome do node)
Insere passcode mostrado no ecrã do T-Deck
Conecta e configura via app

Sair de Bluetooth Programming Mode

Pressiona e mantém ícone Bluetooth no ecrã do T-Deck
Dispositivo reboota de volta ao MUI normal

Importante:
- Bluetooth Programming Mode é para pairing/configuração inicial - Para uso diário com Bluetooth, muda para BaseUI (próxima secção) - MUI não suporta Bluetooth ativo permanentemente (usa mesma API)


Bluetooth - Desativar para Uso Standalone

Quando usar: Operação standalone sem smartphone, economizar bateria, usar MUI com mapas.

Opção 1: Usar MUI (Modo Standalone Recomendado)

MUI (Meshtastic UI) é a interface visual completa mas não suporta Bluetooth simultâneo.

Vantagens: Mapas, teclado, visualização completa de mesh
Desvantagem: Sem Bluetooth para app Para garantir Bluetooth desligado:
Via Meshtastic CLI


Ou via Web Client: - Settings → Bluetooth → Enabled: OFF → Save

Opção 2: BaseUI (Bluetooth Permanente)

Se precisas Bluetooth ativo para app 24/7:

Mudar para BaseUI

Via CLI:

Ou via Web Client: - Settings → Device → Display Type: BaseUI - Save e reboot

BaseUI características: - Interface simples (sem mapas visuais) - Bluetooth funciona permanentemente - Menor consumo recursos - Ideal para controlo via app

Comparação Rápida


Recomendação: Usa MUI para standalone com mapas. Muda para BaseUI se precisas app smartphone sempre conectada.

Troubleshooting
Dispositivo Não Entra em Modo DFU

Sintomas: Ecrã liga normal, computador não deteta dispositivo flash.
Soluções: 1. Repete sequência: Desliga → Trackball → Liga → Aguarda 3s → Liberta 2. Testa cabo: Usa cabo USB-A to USB-C (em vez de C-C) 3. Drivers:
Windows pode precisar drivers CH340/CP210x 4. Tenta 3-5 vezes: Timing é crítico

Bluetooth Não Conecta

Sintomas: App não vê dispositivo, pairing falha.

Soluções:

Se em MUI: - Bluetooth não funciona em MUI normal - Usa Bluetooth Programming Mode: Settings → Reboot → Bluetooth icon - Ou muda para BaseUI
permanentemente

Se Bluetooth disabled:

Se após firmware update: - Pode desativar-se sozinho - Re-ativa via CLI ou web client - Se persistir, rollback para firmware anterior (2.5.18 conhecido funcionar)

iPhone issues: - Alguns firmwares (2.6.x early) têm bugs com iOS - Usa web client USB como alternativa - Atualiza para latest beta firmware

GPS Não Fixa

Sintomas: Posição não atualiza, "No GPS fix" permanente.

Soluções: 1. Vai para exterior: GPS precisa vista céu aberto 2. Aguarda 5-15min: First fix demora (cold start) 3. Verifica configuração:
4. Pins GPIO corretos (T-Deck Plus): - RX: GPIO 34 - TX: GPIO 12 -
Configura via web client: Position → GPS Pins

1. Antena GPS: Alguns modelos têm antena cerâmica integrada, outros precisam externa

Mapas Não Aparecem

Ver secção Troubleshooting Mapas em Mapas Offline.

Dispositivo Não Liga

Soluções: 1. Bateria: Verifica 18650 inserida corretamente (+/- poles) 2. Carrega: Conecta USB-C, aguarda 30min 3. Switch lateral: Confirma posição ON 4.
Reset: Pressiona botão RESET (pequeno buraco lateral)

Firmware Brick / Não Arranca

Soluções: 1. Full Erase + Flash: Modo DFU → Full Erase → Flash firmware beta 2. Testa firmwares diferentes: 2.5.x vs 2.6.x 3. Meshtastic CLI noproto:
4. Factory reset via CLI:

Nodes Não Aparecem / Mesh Vazio

Soluções: 1. Região correta: EU_868 em Portugal 2. Antena conectada: Sem antena = sem TX/RX 3. Frequência slot: Verifica se mesh local usa slot diferente
4. Range: Nodes precisam estar dentro alcance LoRa (~5km urbano, 20km+ rural) 5. Canais: Primary channel deve match com mesh local

Consumo Bateria Alto

Soluções: 1. Desativa Bluetooth: Se não usado 2. GPS update interval: Aumenta para 300s+ (em vez de 120s) 3. Screen timeout: SYM + L para ativar timeout
4. Role: Define como CLIENT (não ROUTER) se móvel
